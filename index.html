Running React Native on Windows

Complete guide to setting up and running React Native apps on Windows

Setting Up React Native on Windows

System Requirements

  • Windows 10 or later
  • Node.js (version 14 or newer)
  • Python 3.x
  • JDK 11 or later
  • Android Studio for Android development

1Install Node.js and Python

Download and install Node.js from the official website. Make sure to install Python 3.x as well (check "Add to PATH" during installation).

node --version # Verify Node.js installation
python --version # Verify Python installation

2Install Java Development Kit (JDK)

Download and install JDK 11 or later from Oracle's website or use OpenJDK.

java -version # Verify Java installation

3Install Android Studio

Download and install Android Studio. During installation:

  • Make sure to install Android SDK
  • Install Android SDK Platform (API Level 31 or higher)
  • Install Android Virtual Device (AVD)

4Configure Android Environment Variables

Set up these environment variables in System Properties:

ANDROID_HOME = C:\Users\YourUsername\AppData\Local\Android\Sdk
PATH = %ANDROID_HOME%\platform-tools;%ANDROID_HOME%\tools;%ANDROID_HOME%\tools\bin

5Install React Native CLI

Open Command Prompt or PowerShell and install the React Native CLI globally:

npm install -g react-native-cli

Creating and Running Your App

1Create a New React Native Project

Create your project using the React Native CLI:

react-native init MyApp

2Navigate to Your Project Directory

cd MyApp

3Start the Metro Bundler

In one terminal window, start the Metro JavaScript bundler:

npx react-native start

4Run Your App on Android

In a separate terminal window, run your app on an Android emulator or device:

npx react-native run-android

Note: Using Android Emulator

Make sure your Android emulator is running before executing the "run-android" command. You can start the emulator from Android Studio's AVD Manager.

Troubleshooting Common Issues

  • If you get "SDK location not found" error, set ANDROID_HOME environment variable
  • If you get "Unable to load script" error, make sure Metro bundler is running
  • For "java not recognized" error, check your JDK installation and JAVA_HOME variable

Development Options on Windows

⬢ Android Development

Windows fully supports Android development for React Native. You can:

  • Use Android Studio emulators
  • Connect physical Android devices via USB debugging
  • Test on various Android API levels

⬢ iOS Development

iOS development on Windows has limitations:

  • You cannot run iOS simulator on Windows
  • Options for iOS development:
    • Use a macOS virtual machine
    • Use a cloud-based macOS service
    • Use Expo for easier cross-platform testing

⬢ Using Expo

Expo provides an easier way to develop React Native apps:

npm install -g expo-cli # Install Expo CLI
expo init MyApp # Create new Expo project
cd MyApp # Navigate to project
expo start # Start Expo development server

Expo allows you to test on both iOS and Android using their app without needing full native setup.

Helpful Commands

react-native run-android --deviceId=YOUR_DEVICE_ID # Run on specific device
react-native log-android # View Android logs
react-native link # Link native dependencies (if needed)
adb devices # List connected Android devices

Debugging Tips

  • Press Ctrl + M in Android emulator to open developer menu
  • Use Chrome DevTools for debugging JavaScript
  • Enable Hot Reloading for faster development
  • Use React DevTools for component inspection

Building for Production

cd android # Navigate to Android directory
./gradlew assembleRelease # Build release APK

The generated APK will be available at android/app/build/outputs/apk/release/app-release.apk

Final Note

While Windows is excellent for Android development with React Native, if you need to develop for iOS, consider using Expo or setting up a macOS environment through a virtual machine or dual boot.
